HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • h1-h6 represent header lines
  • Lists are ul, unordered lists, or ol, ordered lists, with li marking each list item

CSS

  • A margin indicates how much space we want around the ouside of an element
  • A padding indicates how much space we want around the content inside an element
  • An * applies a rule to all elements visible on the page
  • A selector defines the element or attributes to which the declarations will apply
  • Multiple selectors can be combined into a single rule by commas

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name: moves to different branch
  • git add -A: adds changed files to Git staging area
  • git commit -m "description of changes made": commits changes to repository
  • git pull origin main: checks that current branch is up to date before pull request
  • git push origin 'feature branch name': pushes up changes
  • Change requests can then be pulled in GitHub

Javascript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
